# Figure 5 — Public-data capability matrix

**What is plotted.** A 5-by-7 heatmap of capability codes (`direct`, `partial`, `none`) crossing the paper's five core public datasets with seven empirical measurement objects.

**How to read it.** Read each cell as the strength of evidence support a dataset can provide for an object. `direct` means the object is observed in the public source as used. `partial` means proxy or incomplete support. `none` means the object is not supported in the public system.

**Interpretation.** The matrix makes identification boundaries explicit by separating what current public data can measure directly from what remains partial or unsupported.

**Boundary.** This is a documentation-based synthesis figure (rule-driven coding), not an estimated statistical model. Policy recommendations derived from it are design judgments, not causal estimates.

Alt-text: Single-panel figure. The chart is a heatmap matrix with dataset rows and empirical-object columns. Each cell is coded as direct, partial, or none to indicate evidence capability in the current public-data stack.
